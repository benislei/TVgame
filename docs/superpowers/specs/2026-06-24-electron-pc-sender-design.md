# TVGame Electron PC 发送端设计

日期：2026-06-24

## 目标

第一版软件先做 PC 发送端控制台，用 Electron 把已经验证可用的脚本、发送端、输入桥、环境检查和修复能力产品化。目标是让普通用户打开软件后，不需要手动运行多个 bat 或 npm 命令，就能完成环境检查、电视连接、画质选择和开始串流。

本阶段不重写底层串流链路，不替换 Android TV 接收端，不做完整账号、云端、自动更新系统。第一版重点是降低安装和启动门槛，并为朋友试用提供清晰诊断能力。

## 产品形态

采用 Electron 桌面软件，界面语言全部使用中文。

推荐架构为“Electron 管理现有脚本和进程”：

- Electron 主进程负责编排本地命令、进程生命周期、配置读写和日志收集。
- Electron 渲染进程负责中文界面、状态展示、用户确认和错误处理入口。
- 底层继续复用现有能力，包括 `stage2:doctor`、`native:rtp`、InputBridge、ViGEmBus 安装脚本、GStreamer 安装脚本和现有朋友包逻辑。

这样可以最大化复用当前已经实测有效的串流方案，先把体验做成软件，再逐步服务化后台能力。

## 页面结构

### 首次配置向导

首次打开、配置缺失或用户主动重新配置时进入向导。

步骤：

1. 欢迎与准备说明
   - 告诉用户这是电脑发送端。
   - 提醒电视或盒子需要安装并打开 TVGame 接收端。

2. 环境检查
   - 检查 Node/npm。
   - 检查项目依赖。
   - 检查 GStreamer runtime/devel 和核心插件。
   - 检查 H.264/HEVC 编码器能力，包括 NVIDIA、AMD AMF 和系统 Media Foundation。
   - 检查 .NET SDK/Runtime。
   - 检查 ViGEmBus 虚拟手柄驱动。
   - 检查输入桥是否可启动。

3. 一键修复
   - 列出缺失项、影响和推荐处理方案。
   - 用户确认后再执行安装或修复。
   - ViGEmBus 作为手柄支持必装项展示，提示需要管理员权限，可能需要重启。
   - 修复过程展示下载、安装和验证状态。

4. 连接电视
   - 默认自动搜索局域网内打开 TVGame 接收端的电视或盒子。
   - 电视端主动广播设备信息，PC 端显示设备列表。
   - 列表包含设备名、IP、Android 版本、解码器信息、推荐画质档。
   - 支持手动输入 IP 作为兜底，适用于搜不到、跨网段、路由器屏蔽广播等情况。

5. 选择画质
   - 首页和向导只展示预设档位，不开放自定义码率、GOP、端口等参数。
   - 根据编码器能力和设备档案给出推荐档位。
   - 完成后进入日常主屏。

### 日常主屏

日常主屏采用“一键开始 + 状态卡片”结构。

首页展示：

- 当前电视设备和 IP。
- 当前画质档位。
- 性能保护状态。
- 开始串流、停止串流。
- 更换电视。
- 更换画质。
- 环境状态。
- 输入桥状态。
- ViGEmBus 手柄驱动状态。
- 编码器状态。
- 电视端连接状态。

首页不展示完整日志和低层参数。复杂内容进入辅助页面。

### 设备页

用于搜索和管理电视或盒子：

- 自动搜索设备。
- 手动输入 IP。
- 显示设备名、IP、Android 版本、解码器名称和推荐档位。
- 保存最近使用设备。
- 搜索不到时显示可操作建议。

### 画质页

只提供预设档位：

- 720P30：稳定优先，适合电视盒子和弱解码设备。
- 720P60：流畅优先，适合网络稳定但解码一般的设备。
- 1080P30：清晰稳定，适合电视盒子优先尝试。
- 1080P60：高性能，适合手机、高性能电视或盒子。
- HEVC 1080P30：更高压缩效率，适合支持 HEVC 的 Android 11+ 设备。
- HEVC 1080P60：高性能 HEVC 档，适合解码能力较强的 Android 11+ 设备。

高级参数如编码器、码率、GOP、端口、显示器索引保留在高级设置，不放在默认画质页。

### 环境诊断页

用于替代当前分散的环境检查和安装脚本入口：

- 显示每一项检查结果。
- 明确缺失项影响。
- 给出推荐处理方案。
- 用户确认后执行一键修复。
- 修复完成后自动复查。

### 日志页

用于排查朋友机器上的问题：

- 聚合发送端日志。
- 聚合输入桥日志。
- 聚合环境检查和修复日志。
- 支持复制诊断摘要。
- 支持展开详细日志。

## 开始串流流程

点击“开始串流”后，Electron 执行以下流程：

1. 启动前检查
   - 检查当前电视设备或 IP 是否有效。
   - 检查画质档位是否合法。
   - 检查所需编码器是否可用。
   - 检查 GStreamer 是否可用。
   - 检查输入桥和 ViGEmBus 状态。

2. 自动启动输入桥
   - 如果输入桥未运行，自动启动。
   - 如果 ViGEmBus 缺失，提示安装。
   - 如果没有检测到手柄，不阻止串流，只显示“手柄未连接”。

3. 启动发送端
   - 按当前预设档位启动 `native:rtp`。
   - Electron 保存进程句柄。
   - 标准输出和错误输出写入日志页。

4. 运行中监控
   - 显示视频状态、音频状态、输入桥状态和性能保护状态。
   - 允许停止串流。
   - 允许更换画质时先停止当前发送端，再按新档位启动。

5. 停止串流
   - 停止发送端进程。
   - 输入桥默认保持运行，方便用户重新开始；用户退出软件时再关闭或询问。

## 错误处理

错误提示必须使用中文，并提供下一步动作。

典型错误：

- 缺少 H.264 或 HEVC 编码器
  - 提示改用可用档位。
  - 提供“修复环境”入口。

- 电视端未收到画面
  - 提示重新搜索电视。
  - 提示检查电视端 App 是否打开。
  - 提供手动输入 IP。

- 输入桥启动失败
  - 提示检查 .NET。
  - 提供环境修复入口。

- ViGEmBus 缺失
  - 提示安装手柄驱动。
  - 说明可能需要管理员权限和重启。

- 发送端进程异常退出
  - 显示简短原因。
  - 提供重新启动、查看日志、复制诊断摘要。

- 画质档不适合设备
  - 提示切换到更稳定档位。
  - 优先推荐 720P30、1080P30 或 H.264 档。

## 设备发现

设备发现默认使用“电视端广播，PC 端接收”的方式。

电视端广播内容建议包含：

- 应用标识。
- 设备名。
- IP 地址。
- Android API 级别。
- 解码器名称。
- 支持的接收能力。
- 推荐档位或设备档案。

PC 端搜索不到设备时，不把这视为致命错误，显示手动输入 IP 入口。

## 配置存储

Electron 保存以下配置：

- 最近使用的电视设备和 IP。
- 最近使用的画质档位。
- 性能保护开关。
- 是否已完成首次向导。
- 最近一次环境检查摘要。

配置应为本机用户级配置，不要求账号体系。

## 进程与权限

Electron 不静默安装驱动或系统级依赖。

涉及系统修改的动作必须先提示用户确认：

- 安装 GStreamer。
- 安装 Node 运行环境。
- 安装 .NET SDK/Runtime。
- 安装 ViGEmBus。
- 执行需要管理员权限的脚本。

启动发送端和输入桥属于普通运行流程，可以在用户点击“开始串流”后自动执行。

## 打包策略

第一版打包目标是给朋友试用：

- 生成 Windows 桌面安装包或可解压运行包。
- 包含 Electron 应用、现有项目文件、脚本和 Android TV APK。
- 保留“README-朋友试用.md”作为兜底说明。
- 保留原有 bat 入口作为调试备用，但普通用户优先使用 Electron 软件。

后续稳定后再考虑自动更新和后台服务化。

## 测试计划

实现时需要覆盖以下验证：

- Electron 主屏可读取和保存配置。
- 首次向导能从未配置状态进入，并在完成后进入主屏。
- 环境诊断能解析现有检查结果并展示中文缺失项。
- 一键修复必须在用户确认后才执行。
- 设备发现可展示模拟电视端广播信息。
- 手动 IP 可保存并用于启动发送端。
- 每个画质档位生成正确的 `native:rtp` 参数。
- 输入桥未运行时，开始串流会自动启动输入桥。
- 缺少编码器时不会启动失败后只显示原始日志，而是展示中文处理建议。
- 打包产物包含 Electron 应用、接收端 APK 和必要脚本。

## 非目标

本阶段不做：

- 重写 GStreamer 串流管线。
- 重写 InputBridge。
- 重写 Android TV 接收端解码链路。
- 云端账号、远程穿透、STUN/TURN。
- 自动更新系统。
- 4K 正式体验承诺。
- 完整后台 Windows 服务。

## 验收标准

第一版 Electron 软件达到以下标准即可进入朋友试用：

- 新用户可以通过向导完成环境检查、必要修复、电视选择和画质选择。
- 老用户打开主屏后可以一键开始串流。
- 启动串流时自动启动输入桥。
- 画质选择只出现预设档位。
- 电视发现默认自动搜索，手动 IP 可兜底。
- 缺少环境或启动失败时，用户看到的是中文原因和操作按钮。
- 日志页能帮助开发者定位朋友机器上的问题。
